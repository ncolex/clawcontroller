## **CLAWCONTROLLER - RESUMEN TÉCNICO COMPLETO**

### **VISIÓN GENERAL**

**ClawController** es un dashboard de gestión de tareas para agentes de IA OpenClaw. Proporciona una interfaz visual tipo "centro de mando" donde los usuarios pueden organizar, asignar y rastrear el trabajo de múltiples agentes de IA.

**Problema que resuelve:** Los agentes OpenClaw trabajan en sesiones aisladas sin visibilidad centralizada. ClawController añade gestión de workflow estructurado con estados, asignación, y aprobación.

**Sitio web:** https://clawcontroller.com  
**Repositorio:** https://github.com/mdonan90/ClawController  
**Licencia:** MIT

### **ARQUITECTURA TÉCNICA**

#### **Stack Tecnológico**

**Backend:**
- Framework: FastAPI (Python 3.10+)
- Base de datos: SQLite con SQLAlchemy ORM
- WebSockets: Para actualizaciones en tiempo real
- Servidor: Uvicorn (puerto 8000)
- Dependencias principales: fastapi, uvicorn, sqlalchemy, pydantic, websockets

**Frontend:**
- Framework: React 19 + Vite
- UI: Tailwind CSS 4
- State Management: Zustand
- Drag & Drop: @dnd-kit/core, @dnd-kit/sortable
- Iconos: Lucide React
- Utilidades: date-fns, react-router-dom
- Puerto desarrollo: 5001

### **WORKFLOW DE TAREAS**

#### **Estados (Task Lifecycle)**
```
INBOX → ASSIGNED → IN_PROGRESS → REVIEW → DONE
```

**Descripción de estados:**

| Estado | Descripción | Trigger |
|--------|-------------|---------|
| **INBOX** | Sin asignar, necesita triage | Default al crear |
| **ASSIGNED** | Asignado a agente, no iniciado | Asignación manual/auto |
| **IN_PROGRESS** | Agente trabajando activamente | Primera actividad log |
| **REVIEW** | Trabajo completo, necesita aprobación | Agente dice "completed/done" |
| **DONE** | Aprobado y cerrado | Aprobación manual (REVIEW GATE) |

**Regla importante:** Los agentes NO pueden establecer DONE directamente. Deben pasar por REVIEW. Solo el reviewer puede aprobar.

### **INTEGRACIÓN CON OPENCLAW**

#### **Detección de Estado en Tiempo Real**

El backend monitorea los archivos de sesión de OpenClaw:
```python
sessions_dir = ~/.openclaw/agents/{agent_id}/sessions/
```

**Lógica de status:**
- **WORKING:** Sesión activa en últimos 5 minutos
- **IDLE:** Actividad en últimos 30 minutos
- **STANDBY:** Configurado pero sin actividad reciente
- **OFFLINE:** No configurado o unreachable

#### **Monitor de Sesiones (Background Task)**
```python
# Cada 10 segundos:
1. Ejecuta: openclaw sessions list --json
2. Parsea sesiones activas (actualizadas en últimos 60s)
3. Busca tareas ASSIGNED con esos agentes
4. Auto-transición: ASSIGNED → IN_PROGRESS
```

#### **Notificaciones a Agentes**
Cuando ocurren eventos, se notifica vía OpenClaw CLI:
```bash
openclaw agent --agent {agent_id} --message "{mensaje}"
```

**Eventos que notifican:**
- Tarea asignada a agente
- Tarea enviada a REVIEW
- Tarea aprobada (DONE)
- Tarea rechazada (con feedback)

### **AUTO-ASIGNACIÓN**

#### **Configuración**
```python
# backend/main.py
ASSIGNMENT_RULES = {
    "coding": "dev",
    "frontend": "dev",
    "trading": "trader",
    "analysis": "analyst",
    "marketing": "brand",
    "writing": "writer",
    "design": "designer",
}
```

**Funcionamiento:**
1. Al crear tarea con tags, busca match en reglas
2. Primer match gana
3. Asigna automáticamente al agente
4. Status: INBOX → ASSIGNED

Componente raíz. Inicializa store, conecta WebSocket, renderiza layout:

### **INSTRUCCIONES PARA AGENTES (AGENT_INSTRUCTIONS.md)**

#### **Para Lead Agents (Orquestadores)**
```bash
# 1. Crear tarea con assignee
POST /api/tasks {title, description, assignee_id, status: ASSIGNED}

# 2. Log delegación
POST /api/tasks/{id}/activity {agent_id: "main", message: "Delegating to X"}

# 3. Route con contexto fresco
POST /api/tasks/{id}/route

# 4. Aprobar cuando completado
POST /api/tasks/{id}/review {action: "approve", agent_id: "main"}
```

#### **Para Worker Agents**
```bash
# Log progreso mientras trabajas
POST /api/tasks/{id}/activity {agent_id: "dev", message: "Working on X"}

# Cuando termines
POST /api/tasks/{id}/activity {agent_id: "dev", message: "✅ Completed: summary"}
PATCH /api/tasks/{id} {status: "REVIEW"}
```

**Reglas clave:**
- Siempre loguear actividad
- Nunca setting DONE directamente
- Usar REVIEW gate

### **PUNTOS DE INTEGRACIÓN PARA FUSIÓN**

Si quieres fusionar este proyecto con otro, estos son los puntos clave:

#### **1. Base de Datos**
- SQLite en `backend/data/mission_control.db`
- Modelos en `backend/models.py`
- Fácil de migrar a PostgreSQL

#### **2. API Backend**
- FastAPI en `backend/main.py` (3265 líneas)
- Endpoints REST + WebSocket
- Lógica de negocio: notificaciones, auto-asignación, review gate

#### **3. Frontend**
- React + Vite en `frontend/`
- Zustand store en `frontend/src/store/useMissionStore.js`
- 20+ componentes en `frontend/src/components/`

#### **4. Integración OpenClaw**
- Lee config de `~/.openclaw/openclaw.json`
- Monitorea sesiones en `~/.openclaw/agents/{id}/sessions/`
- Notifica vía CLI: `openclaw agent --agent {id} --message "{msg}"`

#### **5. Sistema de Notificaciones**
- Interno: tabla `notifications` + WebSocket
- Externo: OpenClaw CLI subprocess

#### **6. Auto-Assignment**
- Diccionario `ASSIGNMENT_RULES` en `main.py`
- Fácil de extender con lógica custom

#### **7. Recurring Tasks**
- Sistema cron propio
- Tablas: `recurring_tasks`, `recurring_task_runs`
